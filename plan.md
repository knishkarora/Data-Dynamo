# 🚀 EcoLens / NagarWatch — Hackathon Execution Plan

## 🎯 Core Strategy
You are NOT building features.  
You are building a system of accountability:

Citizen reports → system tracks → public sees → pressure builds → corruption reduces.

---

## 🧠 Phase 1 — CORE FEATURES (BUILD FIRST)

### 1. Citizen Reporting System
Problem: No transparent reporting system.

User Flow:
- Open app → map + "Report Issue"
- Upload photo, auto GPS, category
- Submit → appears on public map

UX:
- Floating button
- Map-first UI
- Status badges

---

### 2. Public Feed + Validation
Problem: No trust in reports

User Flow:
- Scroll feed
- Upvote reports
- View on map

UX:
- Clean cards
- Sort by trending/latest

---

### 3. Environmental Map Dashboard
Problem: Data is scattered

User Flow:
- Toggle layers (AQI, Fire, Reports)

UX:
- Map with filters
- Heatmaps

---

### 4. AI Explainer
Problem: People don’t understand data/policies

User Flow:
- Ask questions in chat

UX:
- WhatsApp-like chat UI
- Suggested prompts

---

### 5. Budget Transparency Dashboard
Problem: Corruption in spending

User Flow:
- Select district
- View allocated vs spent

UX:
- Fintech-style cards
- Graphs

---
